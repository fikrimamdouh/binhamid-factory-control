# إعداد الحفظ السحابي وموظف المصنع الذكي

## 1. إنشاء Supabase

1. أنشئ مشروعًا جديدًا في Supabase.
2. افتح **SQL Editor**.
3. انسخ كامل محتوى:
   `supabase/migrations/001_initial_schema.sql`
4. شغّل الملف مرة واحدة.
5. افتح **Project Settings → API** وخذ:
   - Project URL
   - Service Role Key

لا تستخدم Service Role Key داخل المتصفح أو GitHub.

## 2. إعداد متغيرات Vercel

داخل مشروع Vercel:

`Settings → Environment Variables`

أضف القيم التالية إلى Production وPreview:

```text
SUPABASE_URL
SUPABASE_SERVICE_ROLE_KEY
SUPABASE_STORAGE_BUCKET=factory-documents
BINHAMID_ADMIN_TOKEN
```

`BINHAMID_ADMIN_TOKEN` يكون رمزًا عشوائيًا طويلًا، وليس كلمة مرور سهلة. هذا الرمز يُكتب مرة واحدة داخل شاشة «ربط الجهاز» في البرنامج.

بعد إضافة القيم نفّذ Redeploy.

## 3. اختبار الحفظ السحابي

1. افتح البرنامج.
2. افتح تبويب **مركز الاتصال**.
3. افتح **إعداد الخادم**.
4. يجب أن تظهر قاعدة Supabase ورمز الدخول بحالة جاهز.
5. اضغط **ربط الجهاز** وأدخل قيمة `BINHAMID_ADMIN_TOKEN`.
6. اضغط **مزامنة الآن**.
7. افتح البرنامج من متصفح آخر، أدخل الرمز ثم استخدم **استعادة النسخة السحابية**.

عند وجود بيانات محلية، النظام لا يستبدلها إلا بعد تأكيد صريح وينشئ نسخة احتياطية محلية أولًا.

## 4. إنشاء Telegram Bot

1. افتح Telegram وابحث عن `@BotFather`.
2. أرسل `/newbot`.
3. اختر الاسم: `مساعد مصنع بن حامد`.
4. اختر Username ينتهي بـ `bot`.
5. احفظ Token في Vercel فقط تحت:
   `TELEGRAM_BOT_TOKEN`
6. أنشئ سرًا عشوائيًا يحتوي أحرفًا وأرقامًا و`_` أو `-` فقط وضعه في:
   `TELEGRAM_WEBHOOK_SECRET`
7. بعد تشغيل البوت أرسل `/whoami`، ثم ضع رقم حسابك في:
   `TELEGRAM_OWNER_ID`
8. اختيارياً أضف:
   `OPENAI_API_KEY`
   لتفعيل تحويل صوت الميكانيكي إلى نص والرد الصوتي.

بعد إضافة المتغيرات نفّذ Redeploy.

## 5. تسجيل Webhook

1. افتح البرنامج → **مركز الاتصال → إعداد الخادم**.
2. اضغط **تسجيل Webhook**.
3. اكتب Chat ID في اختبار البوت أو أرسل `/whoami` للبوت مباشرة.

Webhook المستخدم:

```text
https://YOUR_DOMAIN/api/telegram/webhook
```

ويتحقق من كل طلب باستخدام `X-Telegram-Bot-Api-Secret-Token`.

## 6. إنشاء المجموعات

أنشئ المجموعات التالية وأضف البوت:

- ورشة مصنع بن حامد
- مالية مصنع بن حامد
- مبيعات وتحصيل البلوك
- مبيعات وتحصيل الخرسانة

اجعل البوت Administrator إذا أردت أن يستقبل كامل رسائل المجموعة، وعطّل Privacy Mode من BotFather عند الحاجة لقراءة المناقشات غير الموجهة له.

بعد إرسال أول رسالة في كل مجموعة ستظهر في:

`مركز الاتصال → المجموعات والمستخدمون`

اعتمد القسم الصحيح لكل مجموعة.

## 7. اعتماد الأشخاص

كل موظف يرسل:

```text
/whoami
```

ثم يظهر في مركز الاتصال. حدّد دوره:

- `admin` مدير النظام
- `manager` المدير
- `accountant` المحاسب
- `mechanic` الميكانيكي
- `block_sales` مبيعات البلوك
- `concrete_sales` مبيعات الخرسانة
- `collector` التحصيل

أي حساب غير معتمد تُحفظ رسائله، لكنه لا يستطيع تنفيذ أوامر أو رؤية تقارير حساسة.

## 8. التجربة الأولى

### Excel

ارفع في مجموعة مناسبة ملفًا باسم مثل:

```text
حركة_يومية_2026-07-15.xlsx
ديزل_2026-07-15.xlsx
رواتب_2026-07.xlsx
```

سيظهر الملف في صندوق الوارد. اضغط **فتح في المستورد** ليعمل مستورد البرنامج الأصلي، ثم راجع الفروقات واعتمدها يدويًا.

### الورشة

أرسل صوتًا أو نصًا:

```text
عطل في المركبة 2345 وفيها تسريب زيت وهي متوقفة
```

النظام يبحث عن اللوحة، يعرض المركبة، ثم يطلب ضغط زر **تأكيد فتح أمر إصلاح**. لا ينشئ الأمر الرسمي اعتمادًا على فهم غير مؤكد.

### المدير

اكتب:

```text
تقارير
```

ثم اختر ملخص اليوم أو الديزل أو الورشة أو المبيعات والتحصيل أو الفروقات.

## 9. سياسات التشغيل

- Excel والصور والصوت تحفظ كملفات أصلية في Storage.
- الملف المكرر يُكتشف ببصمة SHA-256.
- البوت لا يرحّل Excel مباشرة إلى الحسابات.
- الموافقات الرسمية مرتبطة بمعاملة وهوية مستخدم ودور.
- الرواتب ترسل في المجموعة المالية أو المحادثة الخاصة فقط.
- عند فشل قراءة مستند، يبقى في حالة يحتاج إدخالًا يدويًا ولا يتم التخمين.
