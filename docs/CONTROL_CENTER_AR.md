# مركز الرقابة والتسليم

## الغرض

مركز الرقابة ليس شاشة أرقام فقط. هو بوابة قرار تمنع إعلان النظام جاهزًا للتسليم عند وجود خلل مالي أو تشغيلي أو تقني جوهري.

الرابط بعد النشر:

```text
/control-center.html
```

تتطلب الصفحة رمز إدارة صالحًا، ويُحفظ داخل جلسة المتصفح فقط ولا يُرسل إلا في ترويسة Authorization إلى API الإنتاج.

## حالات القرار

### جاهز

لا توجد موانع أو ملاحظات رقابية مفتوحة وفق القواعد المفعلة.

### جاهز تشغيليًا مع ملاحظات

الوظائف الأساسية تعمل، لكن توجد ملاحظات لا تمنع التشغيل اليومي ويجب إغلاقها قبل محضر التسليم النهائي.

### غير جاهز للتسليم النهائي

يوجد مانع واحد على الأقل، مثل:

- قاعدة البيانات أو متغيرات الإنتاج غير مكتملة.
- فرق في مطابقة المبيعات.
- ملفات استيراد فاشلة.
- تجاوز حدود ائتمانية.
- تكاليف غير مصنفة.
- مزامنة أو نسخة احتياطية قديمة.
- تنبيهات حرجة مفتوحة.
- سجل تدقيق فارغ.

## البيانات المعروضة

- المبيعات اليومية وتوزيع البلوك والخرسانة.
- التحصيلات والمبالغ غير الموزعة.
- إجمالي المديونية وتجاوزات الائتمان.
- استهلاك الديزل والتكرارات والحركات غير المرتبطة.
- أوامر الصيانة الحرجة.
- التكاليف غير المصنفة.
- عمر آخر نسخة احتياطية وآخر مزامنة.
- حالة Supabase وTelegram وSchema.
- أحدث أحداث سجل التدقيق.

## حدود المرحلة الأولى

- القرار رقابي آلي ولا يستبدل توقيع الإدارة والمحاسبة والتشغيل.
- الصفحة مستقلة عن الواجهة القديمة لتقليل مخاطر كسر التشغيل.
- ربط زر داخل القائمة الرئيسية يتم بعد نجاح PR واختبار صفحة الإنتاج.
- تسجيل النسخة الاحتياطية داخل جدول `backup_runs` يحتاج مسار إثبات مستقل حتى لا تعتمد الرقابة على وجود Artifact غير مسجل في قاعدة النظام.

## قاعدة التسليم

لا يُكتب في محضر التسليم أن النظام جاهز إلا بعد:

1. حالة مركز الرقابة `ready`.
2. نجاح Production Readiness.
3. نسخة احتياطية حديثة ومثبتة.
4. اختبار متصفح وTelegram وExcel والطباعة.
5. اعتماد المسؤولين على محضر اختبار القبول.
