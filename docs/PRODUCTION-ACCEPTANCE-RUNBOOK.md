# دليل قبول Production والتسليم

## 1. شروط بدء النشر

يمنع النشر إذا لم تتحقق جميع الشروط:

- جميع فحوص Pull Request خضراء.
- تقرير Code Risk يحتوي `P0 findings: 0`.
- آخر Backup ناجح وعمره أقل من 30 ساعة.
- آخر Restore Drill ناجح وعمره أقل من 8 أيام.
- لا Migration أو حذف بيانات ضمن Release Candidate.
- لا Pull Request إصلاح مفتوح يعتمد عليه هذا الإصدار.

## 2. تسجيل نسخة Production

يسجل بعد نجاح Vercel:

| البيان | القيمة |
|---|---|
| Production Commit SHA | — |
| Main Commit SHA | — |
| Vercel Deployment | — |
| وقت النشر بتوقيت الرياض | — |
| Schema Version | — |
| رابط Production | — |
| نتيجة كسر الكاش | — |

يجب أن يتطابق Production Commit SHA مع Main Commit SHA.

## 3. اختبار الإقلاع والجلسة

- فتح من كمبيوتر.
- فتح من هاتف.
- فتح من نافذة خاصة.
- فتح من جهاز جديد.
- الصفحة قابلة للنقر خلال ست ثوانٍ.
- إغلاق وإعادة الفتح دون Reload متكرر.
- ظهور `revision unchanged` عند عدم تغير النسخة.
- ظهور `optional modules loading in idle slices` ثم `optional modules loaded`.
- تسجيل الخروج يمسح الجلسة فقط ولا يحذف بيانات المصنع.
- الدخول التالي ينشئ جلسة جديدة.

## 5. المزامنة

- تعديل سجل من الجهاز الأول.
- فتح الجهاز الثاني وسحب Revision الأحدث.
- محاولة حفظ نسخة قديمة تنتج Conflict Lock ولا تكتب فوق الأحدث.
- السحب يزيل القفل.
- فصل الإنترنت، تنفيذ تعديل محلي، ثم إعادة الاتصال.
- لا تكرار للعملاء أو الأرصدة أو الموظفين.
- ظهور آخر وقت مزامنة ناجحة وحالة الجداول الرئيسية.

## 6. الحضور

- اختيار المصنع لموظف وحفظ الصفحة وإعادة فتحها.
- اختيار المحطة لموظف آخر.
- تسجيل حضور داخل نطاق المصنع.
- تسجيل حضور داخل نطاق المحطة.
- محاولة خارج النطاق تظهر مرفوضة ولا تعتمد كحضور صحيح.
- تسجيل الانصراف.
- مطابقة الموقع والمسافة ودقة GPS والوقت وسبب الرفض في لوحة الإدارة.

## 7. Telegram والصلاحيات

- المستخدم يرى الوحدات المسموحة فقط.
- إخفاء وحدة يمنع الزر والأمر النصي والصوتي وCallback القديم والجلسة المفتوحة.
- وظائف المالك مرفوضة لغير المالك.
- أزرار التأكيد والاعتماد خضراء، الرفض والإلغاء حمراء، الفتح والتنقل زرقاء.
- الضغط مرتين لا ينشئ عمليتين.
- الصوت لا ينفذ عملية مالية أو مخزنية دون تأكيد.

## 8. Excel

- ملف XLSX صحيح يعرض المعاينة وHash.
- ملف XLS صحيح يعرض المعاينة.
- ملف بامتداد غير مسموح يرفض قبل الحفظ.
- ملف فارغ يرفض ولا يغير البيانات القديمة.
- رفع الملف نفسه مرة ثانية يرفض.
- المعاينة تعرض المقبول والمرفوض والمكرر والناقص.
- زر الإلغاء يغلق دون حفظ.

## 9. الطباعة والإرسال

لكل نوع إقرار:

- زر إرسال موجود بجوار زر الطباعة.
- زر الإرسال يشغل زر الطباعة الأصلي.
- الملف يحتوي الشعار والختم والصور والباركود والخطوط.
- عدد الصفحات والبيانات مطابقان للطباعة.
- ورقة فارغة لا ترسل.
- مستند سابق لا يرسل.
- الضغط المكرر يرسل ملفًا واحدًا فقط.

## 10. الاستمرارية

- تنزيل دليل Backup وManifest وChecksum.
- التحقق من SHA-256.
- تنزيل نتيجة Restore Drill.
- مطابقة Schema Version والجداول الحرجة.
- التأكد أن Restore Target ليس Production.
- مراجعة سجل التدقيق والأسرار وImports الفاشلة.

## 11. قرار التسليم

### Ready

- Production مطابق لـMain.
- كل الاختبارات الميدانية ناجحة.
- مطابقة البيانات بلا فروق غير مفسرة.
- Backup وRestore حديثان وناجحان.
- التوقيعات مكتملة.

### Conditional

- لا يوجد فشل في مسار مالي أو تشغيلي أساسي.
- توجد ملاحظات غير مؤثرة، ولكل ملاحظة مسؤول وتاريخ إغلاق.

### Blocked

- Production على كومت قديم.
- بيانات غير متطابقة.
- أداء يمنع الاستخدام.
- فشل مسار مالي أو تشغيلي.
- Backup أو Restore غير متحقق.
