# تكاملات البرنامج ومفاتيح الإنتاج

هذا الملف يحصر أسماء المتغيرات المطلوبة فقط. القيم السرية لا تُكتب في GitHub أو Telegram، وتُحفظ في Vercel Environment Variables وGitHub Repository Secrets بحسب الاستخدام.

## المستودعات المرجعية المختارة

| الوظيفة | المستودع | الاستخدام داخل البرنامج |
|---|---|---|
| قاعدة البيانات والتخزين | `supabase/supabase-js` | مرجع Supabase؛ البرنامج الحالي يستخدم REST وStorage مباشرة من الخادم. |
| الذكاء الاصطناعي والبحث | `openai/openai-node` | الصوت، تحليل الرسائل، ومساعد البحث الحالي عن أسعار المنتجات. |
| الموردون والأماكن | `googlemaps/google-maps-services-js` | مرجع Google Maps Platform وPlaces API. |
| الأسطول الخارجي | `traccar/traccar` | خادم GPS مفتوح المصدر؛ يحتاج خادمًا وحسابًا أو Token وليس مفتاح GitHub. |
| تحويل HTML إلى PDF | `gotenberg/gotenberg` | Chromium API لإنشاء تقارير PDF عربية قابلة للطباعة. |

لا تُنسخ مكتبات إضافية إلى المشروع لمجرد وجود مستودع. التكامل يتم عبر API عندما تكون الخدمة الخارجية مطلوبة، مع إبقاء عدد Serverless Functions ضمن حد Vercel.

## التشغيل الأساسي

| المتغير | مطلوب | الوصف |
|---|---:|---|
| `SUPABASE_URL` | نعم | رابط مشروع Supabase. |
| `SUPABASE_SERVICE_ROLE_KEY` | نعم | مفتاح الخادم فقط؛ ممنوع إرساله للمتصفح. |
| `SUPABASE_STORAGE_BUCKET` | نعم | اسم الحاوية الخاصة، والافتراضي `factory-documents`. |
| `PUBLIC_APP_URL` | نعم | رابط Production العام. |
| `BINHAMID_ADMIN_TOKEN` | نعم | بوابة الإدارة الداخلية. |

## Telegram والحضور والانصراف

| المتغير | مطلوب | الوصف |
|---|---:|---|
| `TELEGRAM_BOT_TOKEN` | نعم | من BotFather. |
| `TELEGRAM_WEBHOOK_SECRET` | نعم | سر التحقق من طلبات Webhook. |
| `TELEGRAM_OWNER_ID` | نعم | Telegram ID لصاحب النظام. |

الحضور والانصراف لا يحتاج API Key مستقلًا. يعتمد على:

1. هوية المستخدم المعتمدة في Telegram.
2. Supabase لحفظ الحركات.
3. إذن الموقع من الهاتف أو WebApp عند استخدام الحضور الجغرافي.
4. المواقع المعتمدة ونطاق السماح المحفوظ داخل النظام.

## الذكاء الاصطناعي والبحث

| المتغير | مطلوب | الوصف |
|---|---:|---|
| `OPENAI_API_KEY` | حسب الاستخدام | الرسائل الصوتية، المساعد الذكي، وبحث أسعار المنتجات بمصادر الويب. |
| `OPENAI_TEXT_MODEL` | اختياري | نموذج النص. |
| `OPENAI_TRANSCRIBE_MODEL` | اختياري | نموذج تحويل الصوت إلى نص. |
| `GOOGLE_PLACES_API_KEY` | حسب الاستخدام | البحث عن الموردين والمتاجر والأماكن. |

يجب تقييد Google API Key على APIs المطلوبة وعلى خادم Production عند الإمكان، مع تفعيل حدود الإنفاق والتنبيهات.

## GPS والأسطول

| المتغير | مطلوب | الوصف |
|---|---:|---|
| `GPS_PROVIDER` | اختياري | الافتراضي `traccar`. |
| `GPS_API_BASE_URL` | عند ربط GPS | رابط خادم المزود. |
| `GPS_API_TOKEN` | أحد الخيارين | Token إذا كان المزود يدعمه. |
| `GPS_API_USER` + `GPS_API_PASSWORD` | بديل Token | حساب خادم GPS. |

وجود Traccar على GitHub لا يوفر خادمًا جاهزًا أو بيانات أجهزة. يلزم تشغيل خادم وربط أجهزة التتبع أو تطبيق العميل.

## PDF وتقارير Excel اليومية

| المتغير | مطلوب | الوصف |
|---|---:|---|
| `PDF_PROVIDER` | نعم للتقارير PDF | القيمة المقترحة `gotenberg`. |
| `PDF_API_URL` | نعم للتقارير PDF | رابط خادم Gotenberg الأساسي أو رابط خدمة JSON متوافقة. |
| `PDF_API_KEY` | اختياري | يُستخدم إذا كانت خدمة PDF محمية. |

عند إرسال ملف التقرير اليومي عبر Telegram:

1. يُحفظ Excel الأصلي في Storage الخاص.
2. تُقرأ مبيعات البلوك والخرسانة والتحصيلات.
3. تُقرأ الأرصدة المعتمدة السابقة من قاعدة البيانات.
4. تُضاف حركة الملف الحالي كمسودة.
5. تُوزع تحصيلات اليوم على أقدم الفواتير وفق FIFO.
6. يُرسل `block-cumulative-YYYY-MM-DD.pdf`.
7. يُرسل `concrete-cumulative-YYYY-MM-DD.pdf`.
8. لا تصبح الحركة نهائية إلا بعد اعتماد الملف من البرنامج.

## المهام والنسخ والاستعادة

| المتغير | مكان الحفظ | الوصف |
|---|---|---|
| `CRON_SECRET` | Vercel/GitHub حسب المهمة | حماية المهام المجدولة. |
| `SUPABASE_DB_URL` | GitHub Secret | اتصال PostgreSQL للنسخ المشفر. |
| `BACKUP_ENCRYPTION_KEY` | GitHub Secret | تشفير النسخ. |
| `RESTORE_DATABASE_URL` | GitHub Secret | قاعدة معزولة لاختبار الاستعادة. |
