# دليل استخدام النظام حسب الوظيفة

## مدير النظام

المسؤوليات:

- اعتماد المستخدمين وإيقافهم.
- مراجعة صلاحيات الأدوار وعدم منح صلاحيات زائدة.
- متابعة Schema وProduction Readiness وGitHub Actions.
- متابعة النسخ المشفرة وRestore Drill وتدوير الأسرار.
- إدارة سجلات الاستمرارية ومحاضر التسليم.

ممنوع:

- إدخال مبيعات أو تحصيلات نيابة عن المحاسب دون تكليف موثق.
- تعديل البيانات مباشرة من Supabase إلا من خلال Migration أو إجراء حادث معتمد.
- عرض أو إرسال قيم الأسرار.

## المدير

المسؤوليات:

- مراجعة مركز الرقابة يوميًا.
- اعتماد أو رفض الاستثناءات الائتمانية.
- إغلاق المخالفات الحرجة بسبب موثق.
- مراجعة الأصول المتوقفة والتكاليف السلبية والتنبيهات.
- توقيع محضر القبول بصفة الإدارة.

لا يملك إقفال الفترة المالية افتراضيًا؛ هذا فصل متعمد بين الإدارة والتنفيذ المالي.

## المحاسب

المسؤوليات:

- استيراد التقرير اليومي ومراجعة المعاينة واعتماده.
- مطابقة الخزن والتحصيلات والمبيعات.
- متابعة كشف العملاء وتخصيص FIFO.
- طلب الاستثناء الائتماني دون اعتماده.
- إقفال وإعادة فتح الفترة بسبب موثق.
- إصدار وتسوية واعتماد العهد حسب السياسة الداخلية.
- توقيع القبول بصفة المالية.

قواعد:

- لا حذف لمبيعات أو تحصيلات معتمدة.
- التصحيح بالعكس أو إعادة فتح الفترة.
- أي تحصيل غير موزع يظهر في مركز الرقابة حتى المعالجة.

## مسؤول مبيعات البلوك أو الخرسانة

- يرى التقارير والبيانات المتعلقة بالعمل.
- لا يعتمد التقرير اليومي.
- لا يغير حدود العملاء.
- لا يعتمد الاستثناء الائتماني.
- يرفع ملاحظة عند اختلاف الكمية أو العميل أو الفاتورة.

## مسؤول الديزل

- يرفع ملف الديزل ويراجع مطابقة اللوحات والأصول.
- لا يضيف أصلًا جديدًا إلا عبر مسؤول الأصول أو مدير النظام.
- يراجع الحركات المكررة وغير المرتبطة.
- يتأكد من استثناء المركبات غير المتوقع استهلاكها للديزل.

## الميكانيكي أو مسؤول الصيانة

- يفتح البلاغ ويسجل التشخيص والحالة والتكلفة والمرفقات.
- لا يغلق أمرًا قبل إكمال التشخيص.
- إذا تجاوزت التكلفة 100 ر.س يلزم مرفق قبل الإغلاق.
- يربط الأمر بالأصل الصحيح.
- يختبر المركبة قبل تحويل الحالة إلى مكتمل ثم مغلق.

## الموارد البشرية

- تدير ملف الموظف ومستنداته وتواريخ الانتهاء.
- تراجع الهوية والإقامة والرخصة والتأمين والعقد.
- لا تعتمد استثناءات ائتمانية ولا تغلق فترات مالية.
- تتابع تنبيهات 60 و30 يومًا والانتهاء.

## السائق

- يسجل عبر Telegram ويرفع المستندات المطلوبة.
- لا يستخدم مركبة غير مسندة له.
- يسجل الديزل والعداد والموقع والإيصال.
- يفتح بلاغ عطل ولا يغلق أمر الصيانة.
- لا يطلع على البيانات المالية أو بيانات الموظفين الأخرى.

## جهاز المصنع

جلسة الجهاز تلقائية وآمنة وتسمح بـ:

- تشغيل البرنامج والحفظ السحابي التشغيلي.
- عرض مركز الرقابة والحوكمة.
- استيراد واعتماد التقرير اليومي حسب المستخدم المسجل.

لا تسمح بـ:

- إقفال الفترات.
- اعتماد تجاوز ائتماني.
- إدارة الأصول أو المستندات.
- اعتماد العهد.
- تسجيل اختبار استعادة أو توقيع تسليم.

هذه الإجراءات تحتاج مستخدمًا معتمدًا وصلاحية صريحة.

## دورة العمل اليومية

1. فحص المزامنة ونسخة Backup الحديثة.
2. مراجعة الملفات الواردة من Telegram.
3. اعتماد التقرير اليومي بعد المعاينة.
4. مراجعة مطابقة المبيعات والتحصيلات.
5. معالجة الحركات غير المرتبطة والمخالفات.
6. مراجعة أوامر الصيانة والأصول المتوقفة.
7. طباعة أو تصدير ملخص الإدارة.

## دورة العمل الأسبوعية

- مراجعة الديون والحدود والاستثناءات.
- مراجعة الديزل لكل أصل والانحراف.
- مراجعة الصيانة المفتوحة ومدد التوقف.
- مراجعة المستندات التي تنتهي خلال 60 يومًا.
- التأكد من نجاح Restore Drill.

## دورة العمل الشهرية

- إكمال تصنيف التكاليف ومراجعة هامش البلوك والخرسانة.
- تسوية العهد والتحصيلات غير الموزعة.
- إقفال الفترة المالية بسبب موثق.
- إصدار التقرير الشهري ومحضر المراجعة.
- مراجعة المستخدمين والصلاحيات والأسرار المستحقة للتدوير.
