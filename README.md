# نظام رقابة مصنع بن حامد

نظام تشغيل ورقابة للمصنع يعمل محليًا داخل المتصفح، مع طبقة اختيارية للحفظ السحابي المركزي ومركز اتصال عبر Telegram Bot.

## ما تم بناؤه

- الحفاظ على البرنامج الأصلي كاملًا داخل `legacy.html` دون إعادة كتابة وظائفه.
- Loader صغير في `index.html` يحمّل النسخة الأصلية ثم يضيف طبقة السحابي ومركز الاتصال.
- حفظ Local-first: الحفظ المحلي يظل فعالًا حتى عند انقطاع الإنترنت.
- مزامنة سحابية بتعقب Revision ومنع الاستبدال الصامت عند وجود نسخة أحدث.
- قاعدة بيانات Supabase تشمل المستخدمين، المجموعات، الملفات، أوامر الإصلاح، عروض الأسعار، الاعتمادات، الفواتير، الفروقات وسجل التدقيق.
- Telegram webhook يستقبل النصوص، Excel، الصور، المستندات والصوت.
- تحويل الصوت العربي إلى نص والرد الصوتي عند إضافة `OPENAI_API_KEY`.
- صندوق وارد داخل البرنامج؛ ملف Excel الوارد لا يُرحّل مباشرة بل يُفتح في نفس مستورد الحركة أو الديزل الموجود بالبرنامج.
- تعريف مجموعات الورشة والمالية والبلوك والخرسانة وصلاحيات المدير والمحاسب والميكانيكي والمبيعات.
- أوامر تقارير للمدير وفتح بلاغ عطل بعد تأكيد المركبة.

## مبدأ الأمان

- لا توجد مفاتيح سرية داخل المستودع.
- Supabase Service Role وTelegram Token وOpenAI Key تحفظ في Vercel Environment Variables فقط.
- كل جداول Supabase مفعّل عليها RLS ولا توجد سياسات وصول عامة.
- الاعتماد المالي لا يُنفذ من كلام عابر؛ يجب Callback/زر مرتبط بمعاملة محددة ومستخدم معتمد.
- البيانات المحلية لا تُستبدل تلقائيًا عند وجود بيانات على الجهاز.

## التشغيل

نفّذ الخطوات الموجودة في:

- `docs/SETUP_AR.md`
- `supabase/migrations/001_initial_schema.sql`
- `.env.example`

## الاختبارات

```bash
npm install
npm test
```

## بنية المشروع

```text
index.html                         loader
legacy.html                        البرنامج الأصلي الكامل
assets/cloud-control.css           واجهة مركز الاتصال والسحابي
assets/cloud-control.js            المزامنة وربط الواجهة
api/state.js                       حفظ واستعادة الحالة المركزية
api/dashboard.js                   لوحة الوارد والاعتمادات
api/telegram/webhook.js            موظف Telegram الذكي
api/telegram/register.js           تسجيل Webhook
api/admin/*                        اعتماد المجموعات والمستخدمين
api/imports/*                      تنزيل ملفات الوارد وتحديث حالتها
supabase/migrations/*              مخطط قاعدة البيانات
```

## حدود النسخة الحالية

الكود جاهز للنشر، لكن الحفظ السحابي والبوت يظلان في وضع «ينتظر الإعداد» إلى أن يتم إنشاء مشروع Supabase، تشغيل ملف SQL، وإضافة متغيرات Vercel وإنشاء البوت من BotFather.
